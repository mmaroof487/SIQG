Request Signing (HMAC)
Mediumunique
Require API requests to be signed with an HMAC-SHA256 signature using a shared secret. Prevents replay attacks — a captured request can't be replayed because the timestamp is part of the signature. This is exactly how AWS API authentication works.
Pitch: "My API uses HMAC request signing — same model as AWS Signature V4, prevents replay attacks."
🧬
Query Complexity Scoring
Mediumunique
Score every query for complexity — number of JOINs, subqueries, wildcards, missing WHERE clauses. Assign a score and block queries above a threshold unless the user is an admin. A SELECT * with 3 JOINs and no WHERE on a 10M row table is a DoS vector.
Pitch: "My gateway scores query complexity and rejects queries that would cause DB strain."

Automatic LIMIT Injection
Easyunique
If a SELECT query has no LIMIT clause, automatically inject LIMIT 1000 before execution. Prevents accidental full table scans returning millions of rows. Show the injected LIMIT in the query diff viewer. Simple regex/AST check, huge safety impact.
Pitch: "My gateway auto-injects LIMIT clauses — no unbounded SELECTs ever hit the database."

Result Size Capping
Easy
Even with a LIMIT, a row with 50 columns of JSON blobs can return 100MB. Cap the total response size (e.g. 5MB). If exceeded, return a truncated result with a warning header. Prevents memory exhaustion attacks on your gateway.
Pitch: "Response size is capped at the gateway — prevents memory exhaustion from large result sets."

Honeypot Table Detection
Mediumvery unique
Create a fake table (e.g. secret_keys, admin_passwords) that no legitimate app should ever query. If any query touches it — instant alert, block the user. This is a real security pattern called a honeypot. No other college project has ever done this. Extremely memorable in interviews.
Pitch: "I planted honeypot tables — any query that touches them triggers an immediate security alert."

Pitch: "Access control goes to the column level — roles define exactly which fields they can read."

Retry with Exponential Backoff
Easy

Pitch: "Transient failures trigger exponential backoff retries — same pattern used in every cloud SDK."

Query Plan Regression Detection
Hardvery unique

Pitch: "My gateway detects query plan regressions — it alerts when a previously-indexed query starts doing full table scans."
